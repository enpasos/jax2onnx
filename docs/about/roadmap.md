# Roadmap

## Planned

* Broaden coverage of JAX, Flax NNX/Linen, and Equinox components.
* Expand SotA example support for vision and language models.
* Improve support for **physics-based simulations**.
* Support for **[MaxDiffusion](https://github.com/AI-Hypercomputer/maxdiffusion)**.


## Current Version

### **jax2onnx 0.12.1** 

* **Custom ONNX graph I/O naming (`#186`, `#188`):** Added configurable `input_names` / `output_names` in `to_onnx(...)`, wired through conversion API + UI, and covered by dedicated converter/example tests.
* **Coverage expansion wave (`#189`):** Added a broad set of new primitive plugins and hardening across `jax.lax`, `jax.nn`, `jax.numpy`, `jax.random`, Flax Linen/NNX, and `dm_pix` (including new `space_to_depth` and `prelu` support).
* **Additional primitive coverage (`#195`):** Added `jax.lax.atan2` and `jax.lax.complex` lowerings plus associated test coverage.
* **MaxText export unblocking (`#194`):** Added `jax.lax.triangular_solve` lowering and ORT-compat fixes to unblock newer MaxText model exports; updated MaxText guide/integration behavior for current upstream structure.
* **Converter/runtime robustness:** Extended numeric-validation guardrails and reduce-op structural coverage tests; fixed multiple shape/metadata/runtime edge-cases in reduce, sort/topk, reshape, scatter, and related lowering paths.
* **Operator-coverage transparency:** Added autogenerated ONNX operator coverage matrix and generation script (`docs/user_guide/onnx_operator_coverage.md`, `scripts/generate_onnx_operator_coverage.py`).
* **Dependency refresh:** Updated supported stack and lockfile (JAX `0.9.0.1`, Flax `0.12.4`, Equinox `0.13.4`, onnx-ir `0.1.16`, onnx `1.20.1`, onnxruntime `1.24.2`).
* **Documentation polish:** Updated roadmap/architecture styling and added license badge updates in `README.md` and `docs/index.md`.
